Politique de sécurité

Ce site est conçu pour être plus sûr, même s'il reste une application web statique sans serveur dédié.

Ce que nous avons protégé

  • Les mots de passe des comptes sont hachés avant d'être stockés dans le navigateur.
  • Le compte administrateur est masqué et nécessite un paramètre secret pour se connecter.
  • Les informations affichées sur l'administration sont nettoyées pour éviter les injections HTML.
  • La page de connexion n'expose pas le mot de passe de l'utilisateur après connexion.

Ce qui reste limité

Ce site est une application frontale statique. Pour être vraiment protégé contre les attaques, il faut :

  • un hébergement sécurisé (HTTPS),
  • un serveur avec base de données,
  • un backend qui contrôle l'accès et vérifie les entrées utilisateur,
  • des sauvegardes régulières et une protection antivirus sur l'ordinateur.

Conseils pour les utilisateurs

  • Utilise un mot de passe unique et suffisamment long.
  • Ne partage jamais ton mot de passe.
  • Si tu veux un vrai espace sécurisé, passe par un vrai backend et un serveur HTTPS.
